# 公开发布与安全边界

## 发布目标

- 仅发布通用方法、模板、脚本和检查清单。
- 不发布任何真实项目材料、客户材料、未公开技术文本、生成结果样稿或可反推来源的信息。

## 发布前必须检查

1. 技能目录中不包含任何 `.docx`、`.pdf`、截图、导出结果、临时文件或测试样稿。
2. 技能文本中不包含绝对路径、内部文件名、日期版次、真实项目名、具体对象名称、IP、邮箱、手机号或其他可识别信息。
3. 技能内容只保留通用规则，不保留来自某次具体任务的术语映射、案例表述、问题清单或经验性“定制答案”。
4. 所有示例文件名都使用通用名称，如 `disclosure.md`、`template.docx`。
5. 若脚本依赖本地工具，只说明调用方式，不绑定私人目录结构。
6. 运行以下审计命令并人工复核输出：

```bash
python3 scripts/public_release_audit.py .
```

## 使用时的安全边界

- 仅在用户明确拥有处理授权时，协助整理技术材料。
- 若材料含个人敏感信息、商业秘密、未公开专利内容、攻击细节、规避监管信息或其他高风险内容，先要求用户脱敏或改写为抽象描述。
- 对违法违规、危害人身安全、攻击实施、逃避监管、定向伤害或不当收集敏感信息的请求，直接拒绝处理，不生成任何正文、提纲、示例或部分结果。

## 固定拒绝语

- 如触发拒绝条件，仅输出：
  `基于合规与安全要求，当前请求不予处理。`

## 建议的公开仓库策略

- 单独发布技能目录，不与实际项目工作区混放。
- 使用单独仓库或子目录作为公开源。
- 在发布前逐文件人工复核一次。
- 对外只共享技能路径，不共享工作区其余文件。
