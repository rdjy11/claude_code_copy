# Skill 质量评审



## 概述



基于 Darwin 8 维度 Rubric 对任意 Claude Code Skill 进行标准化质量评估，输出百分制评分报告。



**核心原则**：Skill 不是文档，是塑造 Agent 行为的代码。评审标准是"Agent 读完后能否正确、自主、高质量地执行任务"。



## 评审触发条件



- 用户要求评审某个 Skill

- 安装新 Skill 前的质量评估

- 自研 Skill 的上线前审查

- 批量评审多个 Skill 的横向对比



## 评审工作流



### Step 1: 获取 Skill 内容



必须读取 Skill 的**全部文件**：

1. SKILL.md（主文件）

2. references/ 目录下所有文件

3. scripts/ 目录下所有文件

4. 其他附属文件（templates/、examples/ 等）



**不可只读 SKILL.md 就评分**——资源整合度(D6)需要看到完整目录结构。



### Step 2: 逐维度评分



按照 8 个维度依次评分，每个维度必须给出：

- 分数（整数）

- 1-2 句评价依据



详见 `references/darwin-rubric.md`。



### Step 3: 计算总分与等级



| 等级 | 分数区间 | 含义 |

|------|---------|------|

| S 优秀 | 90-100 | 标杆级，可直接推广 |

| A 良好 | 80-89 | 专业级，小幅优化即可 |

| B 合格 | 70-79 | 可用但有明显短板 |

| C 待改进 | 60-69 | 需要显著改进才能生产使用 |

| D 需重写 | <60 | 不建议使用，需要重新设计 |



### Step 4: 输出报告



必须使用标准报告格式（见下方"输出格式"）。

## 检查点



**CP1: 内容完整性确认**

- 必须暂停确认：是否已读取全部文件？是否遗漏了 references/ 或 scripts/？



**CP2: 评分一致性校验**

- 评分完成后自检：总分是否 = 各维度之和？各维度分数是否在满分范围内？



## 边界条件



| 情况 | 处理 |

|------|------|

| Skill 只有一个 SKILL.md 无附属文件 | D6 资源整合度最高给 2 分（除非文件本身极短<100行且功能单一） |

| Skill 是框架/套件（多个子skill） | 取各子skill加权均值，同时评估框架整体设计 |

| Skill 是纯英文，评审环境为中文 | 在 D8 中扣分（适用性受限），但不影响其他维度 |

| Skill 无法获取完整源码 | 注明"部分评估"，D6 和 D8 给保守分 |

| 用户要求批量评审 | 输出汇总表（同文档中的表格格式）+ 每个Skill的简评 |



## 反模式



- 只看 SKILL.md 开头就打分（必须通读全文）

- 凭"感觉"给分不给依据（每个维度必须有具体证据）

- 对知名项目加分、对无名项目扣分（只看内容不看名气）

- 把"行数多"等同于"质量高"（简洁但完整 > 冗长但松散）

- 忽略实际可执行性只看理论完整性（D8 占 25 分，是最大权重）
