# RTK (Rust Token Killer) - Token-Optimized Commands

## Golden Rule

**Always prefix commands with `rtk`**. If RTK has a dedicated filter, it uses it. If not, it passes through unchanged. This means RTK is always safe to use.

**Important**: Even in command chains with `&&`, use `rtk`:
```bash
# ❌ Wrong
git add . && git commit -m "msg" && git push

# ✅ Correct
rtk git add . && rtk git commit -m "msg" && rtk git push
```

# Superpowers Integration

Superpowers (`superpowers:*` skills) is a **software development methodology framework**. It enforces a mandatory workflow for code tasks: brainstorm → design approval → worktree isolation → plan → execute → review → finish.

## Golden Rule: Superpowers for Code, Other Skills for Knowledge Work

| 场景 | 该用什么 | 说明 |
|------|---------|------|
| 写代码、修bug、加功能、做feature | **Superpowers workflow** | 走 brainstorm → plan → tdd → execute → review 全流程 |
| 分析论文、写专利、拆书、蒸馏人物 | **原有技能** (xray/academic-paper/cangjie/nuwa/patent) | Superpowers 不涉及这些领域 |
| 代码审查 | **Superpowers 流程** requesting-code-review → code-reviewer agent | system `review`/`security-review` 做补充审查 |
| 调试问题 | **Superpowers** systematic-debugging | 根的调试方法论 |
| 模糊需求/决策建议 | **区分**：软件相关→brainstorming；思维框架相关→nuwa | 不确定时间我 |

## Conflict Rules

1. **Superpowers 技能优先于同领域的系统内置技能**。例如代码审查优先走 `requesting-code-review` + `code-reviewer` agent，然后用 `review` 做补充。
2. **非代码任务不走 Superpowers gate**。当你调用 nuwa/cangjie/xray/academic-paper/composer/patent/boss-perspective 时，不需要经过 brainstorming → design approval 流程。
3. **边界不清时问用户**：如果你不确定当前任务属于"软件开发"还是"知识工作"，直接问我。
